# Godot 脚本学习文档（GDScript 专项）

## 1. 目标
学完本文件后，你应该能：
- 独立编写一个 2D 小游戏主逻辑脚本
- 快速读懂现有脚本并安全修改
- 出现报错时，能自己定位并修复

---

## 2. 先建立正确认知
- 脚本是挂在节点上的行为代码。
- 同一个场景可以有多个脚本协作。
- 好脚本不是“短”，而是“职责清晰”。

建议把脚本按职责拆分：
- `player.gd`：角色移动和状态
- `game_manager.gd`：分数、流程、重开
- `ui_hud.gd`：界面显示

---

## 3. 语法最小必会集
### 3.1 变量与类型
```gdscript
var score: int = 0
var speed: float = 320.0
const MAX_LIFE := 3
```

### 3.2 条件与循环
```gdscript
if life <= 0:
	game_over = true

for i in range(items.size()):
	print(i)
```

### 3.3 生命周期函数
- `_ready()`：节点进入场景后调用一次
- `_process(delta)`：每帧调用（非物理）
- `_physics_process(delta)`：固定物理帧调用
- `_draw()`：自定义绘制

---

## 4. 以 `catch_game.gd` 理解完整循环
### 输入
读取左右方向键，控制挡板移动。

### 生成
按 `spawn_gap` 定时生成下落球。

### 更新
每帧更新球位置，离开屏幕则扣血。

### 碰撞
挡板和球重叠时加分并移除球。

### 状态
`game_over = true` 后暂停流程，按 `ui_accept` 重开。

这个结构就是典型的“输入 -> 更新 -> 判定 -> 渲染 -> 状态切换”。

---

## 5. 常见错误与修复
### 错误 1：脚本类型不匹配
报错：脚本是 `Node2D`，却挂到 `Node`。
修复：根节点改成 `Node2D`。

### 错误 2：缩进错误
GDScript 对缩进严格敏感。
修复：统一使用 Tab 或 4 空格，不要混用。

### 错误 3：输入无效
原因：`InputMap` 未定义动作。
修复：在项目设置里补齐动作绑定。

### 错误 4：逻辑能跑但看不到效果
原因：对象在屏幕外或层级被遮挡。
修复：检查坐标、`z_index`、相机范围。

---

## 6. 调试方法（建议固定流程）
1. 先看报错第一行，不要跳读。
2. 在关键变量处加 `print()`（如 `score`、`life`）。
3. 用断点逐帧看变量变化。
4. 修复后立刻回归测试同路径。

---

## 7. 从“能跑”到“好维护”
- 函数不要过长，超过 40~60 行就考虑拆分。
- 魔法数字抽成常量（例如速度、间隔、血量）。
- 先定义状态，再写状态切换条件。
- UI 更新通过信号触发，不要到处直接改节点。

---

## 8. 练习清单（按顺序）
1. 新增难度曲线：分数每增加 10，`spawn_gap` 减少 0.05。
2. 增加连击系统：连续接球给额外分数。
3. 增加道具球：回血球、减速球。
4. 加入暂停菜单与继续功能。
5. 把分数保存到本地文件并显示最高分。

---

## 9. 验收标准
- 你能不看教程独立解释 `catch_game.gd` 的每个函数。
- 你能在 30 分钟内新增一个小玩法并稳定运行。
- 你能在出现报错时独立定位到具体代码行。

达到这三个标准，说明你已具备 Godot 脚本实战基础。
