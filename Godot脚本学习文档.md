# Godot 脚本学习文档（GDScript）

## 1. 学习目标
- 能独立写出一个可运行的 2D 小游戏脚本。
- 看懂并修改现有脚本中的变量、逻辑和绘制代码。
- 遇到报错时能快速定位并修复。

## 2. 基础概念
- 节点（Node）：游戏对象的基础单位。
- 场景（Scene）：节点的组合，可保存复用。
- 脚本（Script）：挂在节点上，控制节点行为。
- 常见类型：
  - `Node`：通用基础节点。
  - `Node2D`：2D 节点，可用于绘制和位置控制。
  - `Control`：UI 节点（按钮、文本等）。

## 3. GDScript 基础语法
- 变量：`var score = 0`
- 函数：
  - `func _ready():` 初始化时执行一次
  - `func _process(delta):` 每帧执行
  - `func _draw():` 自定义绘制
- 条件：`if / elif / else`
- 循环：`for i in range(...)`
- 容器：
  - 数组：`var balls = []`
  - 字典：`{"p": Vector2(...), "v": 200.0}`

## 4. 你当前小游戏脚本结构
脚本文件：`catch_game.gd`

- 顶部变量区：
  - 玩家参数（位置、速度、宽高）
  - 小球列表与生成参数
  - 分数、生命、游戏状态
- `_ready()`：
  - 初始化随机数
  - 初始化玩家位置
- `_process(delta)`：
  - 读取输入（左右移动）
  - 定时生成小球
  - 更新小球下落
  - 碰撞检测与得分/扣血
- `_draw()`：
  - 绘制挡板
  - 绘制下落小球
- `_restart()`：
  - 重置分数、生命、小球和位置

## 5. 常见报错与处理
- 报错：`Script inherits from native type 'Node2D' ... object of type: 'Node'`
  - 原因：脚本是 `Node2D`，挂在了 `Node` 上。
  - 处理：把节点换成 `Node2D`，或脚本改为 `extends Node`（不推荐本项目）。

- 报错：首行/顶层缩进错误
  - 原因：`extends`、`var` 前多了空格或隐藏字符。
  - 处理：顶层内容必须顶格写，只有函数体内部缩进。

- 报错：语法错误（常见于粘贴）
  - 原因：混入不可见字符（如方块符号）。
  - 处理：全选删除后重新粘贴，或直接导入 `.gd` 文件。

## 6. 编辑器设置建议（中文版）
- 路径：`编辑器 -> 编辑器设置`
- 搜索：`缩进`、`粘贴`
- 关闭：
  - 粘贴时自动缩进
  - 自动缩进（按需）

## 7. 实战练习（按顺序）
1. 修改 `spawn_gap`（如 0.8 -> 0.5），观察难度变化。
2. 新增 `Label` 显示分数和生命值。
3. 实现难度递增：分数越高，下落速度越快。
4. 加入开始菜单和游戏结束界面。
5. 加音效（接住球、漏球、结束）。

## 8. 编写规范
- 顶层代码不缩进。
- 函数内统一用 Tab 或 4 空格，保持一致。
- 变量命名清晰：`player_speed`、`spawn_gap`。
- 一次只改一小块，改完立即运行验证。

## 9. 推荐学习节奏
- 每天 20-30 分钟：
  - 10 分钟读代码
  - 10 分钟改一个功能
  - 10 分钟修一个小 bug

完成上面练习后，你就具备独立写简单 Godot 2D 脚本的能力。
